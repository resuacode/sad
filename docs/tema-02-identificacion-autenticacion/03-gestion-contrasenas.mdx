# Gestión de Contraseñas

## Introducción

La gestión adecuada de contraseñas es fundamental para la seguridad de cualquier sistema.

## Características de una contraseña segura

### Complejidad
- **Longitud mínima**: 12 caracteres
- **Combinación de caracteres**:
  - Letras mayúsculas (A-Z)
  - Letras minúsculas (a-z)
  - Números (0-9)
  - Símbolos especiales (!@#$%^&*)

### Exclusividad
- Una contraseña única por cuenta
- No reutilizar contraseñas anteriores
- No usar información personal

## Políticas de contraseñas

### Elementos de una política

```markdown
- Longitud mínima: 12 caracteres
- Complejidad: Al menos 3 tipos de caracteres diferentes
- Caducidad: 90 días (para sistemas críticos)
- Historial: No reutilizar las últimas 10 contraseñas
- Bloqueo: Después de 5 intentos fallidos
```

### Excepciones y casos especiales
- Cuentas de servicio
- Cuentas administrativas
- Sistemas legacy

## Gestores de contraseñas

### Ventajas
- Generación de contraseñas seguras
- Almacenamiento cifrado
- Sincronización entre dispositivos
- Detección de contraseñas débiles

### Herramientas recomendadas
- **Empresariales**: 1Password Business, Bitwarden Enterprise
- **Personales**: Bitwarden, KeePass, LastPass

## Ataques a contraseñas

### Tipos de ataques

#### Fuerza bruta
Probar todas las combinaciones posibles sistemáticamente.

#### Ataque de diccionario
Usar listas de contraseñas comunes.

#### Ingeniería social
Obtener contraseñas mediante manipulación psicológica.

#### Credential stuffing
Usar credenciales obtenidas de filtraciones anteriores.

### Medidas de protección
- Implementar bloqueo de cuentas
- Usar CAPTCHA
- Monitorizar intentos de acceso
- Implementar MFA
