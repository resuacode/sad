# Tema 7: Alta Disponibilidad

## Introducción

La alta disponibilidad se refiere a la capacidad de un sistema de mantenerse operativo durante el mayor tiempo posible.

## Métricas importantes

### Disponibilidad
Porcentaje de tiempo que un sistema está operativo.

### MTBF (Mean Time Between Failures)
Tiempo medio entre fallos.

### MTTR (Mean Time To Repair)
Tiempo medio de reparación.

### RTO (Recovery Time Objective)
Tiempo máximo aceptable para recuperar un servicio.

### RPO (Recovery Point Objective)
Cantidad máxima de datos que se puede perder.

## Niveles de disponibilidad

- **99%**: 3.65 días de downtime al año
- **99.9%**: 8.76 horas de downtime al año
- **99.99%**: 52.56 minutos de downtime al año
- **99.999%**: 5.26 minutos de downtime al año

## Técnicas de alta disponibilidad

- Redundancia
- Clustering
- Load balancing
- Failover automático
- Replicación de datos
