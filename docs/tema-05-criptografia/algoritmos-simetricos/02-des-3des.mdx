# DES y 3DES

## DES (Data Encryption Standard)

### Historia
- Desarrollado en los años 70 por IBM
- Adoptado como estándar en 1977
- **Estado actual**: Considerado inseguro

### Características
- **Tamaño de clave**: 56 bits (efectivos)
- **Tamaño de bloque**: 64 bits
- **Rondas**: 16

### Vulnerabilidades
- **Fuerza bruta**: Clave demasiado corta
- **Criptoanálisis diferencial**: Ataques teóricos

## 3DES (Triple DES)

### ¿Por qué 3DES?
Extensión de DES para aumentar la seguridad aplicando DES tres veces.

### Modalidades

#### EDE (Encrypt-Decrypt-Encrypt)
```
C = E_K3(D_K2(E_K1(P)))
```

#### Configuraciones de claves
- **3DES-2KEY**: K1 = K3, K2 diferente
- **3DES-3KEY**: Las tres claves son diferentes

### Seguridad
- **Longitud efectiva**: 112 bits (2KEY) o 168 bits (3KEY)
- **Estado**: Deprecado para nuevas implementaciones

## Comparación con AES

| Aspecto | DES | 3DES | AES |
|---------|-----|------|-----|
| Tamaño de clave | 56 bits | 112/168 bits | 128/192/256 bits |
| Seguridad | ❌ Inseguro | ⚠️ Deprecado | ✅ Seguro |
| Rendimiento | Rápido | Lento | Muy rápido |
| Uso actual | No recomendado | Legacy | Recomendado |

## Migración de sistemas legacy

### Estrategias
1. **Reemplazo directo**: Cambiar directamente a AES
2. **Migración gradual**: Coexistencia temporal
3. **Wrapper de compatibilidad**: Capa de abstracción

### Consideraciones
- **Compatibilidad**: Sistemas antiguos
- **Certificaciones**: Algunos sistemas requieren 3DES
- **Rendimiento**: Impacto en sistemas críticos
